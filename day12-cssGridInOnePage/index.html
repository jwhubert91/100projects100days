Introduction

CSS Grid Layout is the most powerful layout system available in CSS. It is a 2-dimensional system, meaning it can handle both columns and rows, unlike flexbox which is largely a 1-dimensional system. You work with Grid Layout by applying CSS rules both to a parent element (which becomes the Grid Container) and to that element's children (which become Grid Items).

(Source: css-tricks.com)

CSS grid can create more asymmetrical layouts than the previous grid and layout options like CSS floats. It also allows for more standardized code that works across browsers. This is in contrast to relying on specific browser hacks or complicated workarounds.

One issue with using floats in CSS is if content gets added to one portion of the page it could disrupt the flow of the page and break the layout. This is due to the varying heights for layout elements. Though CSS flexbox supports flexible layouts and provides the flexibility of creating complex layouts, it fails when the need for creating responsive layouts in 2-dimensional space arises. That is where CSS grid layout shines.

(Source: Wikipedia.org)

CSS is a cornerstone technology of the web, alongside HTML and Javascript. CSS is designed to enable the separation of presentation and content, including layout, colors, and fonts. This separation can improve content accessibility, provide more flexibility and control in the specification of presentation characteristics, enable multiple web pages to share formatting by specifying the relevant CSS in a separate .css file, and reduce complexity and repetition in the structural content.

  • Styling text and images
  • Setting the background color and border of a div

  • The basics of color notation in CSS (named colors, hex codes, rgb)

Grid vs Flexbox

The major difference between these technologies is that CSS Grid is made for two-dimensional layout while Flexbox is for one. This means Flexbox can work on either row or columns at a time, but Grids can work on both.

When working on either element (row or column), you are most associated with the content. Flexbox, here, gives you more flexibility. HTML markup and CSS will be easy to manage in this type of scenario.

However, when working with the overall web page layouts, CSS Grid does away with the nightmares of coding. It gives you more flexibility to move around the blocks irrespective of your HTML markup.

Quick comparison:
Element Grid Flexbox
Dimension Two dimensional One dimensional
Support type Layout-first Content first
Page responsiveness Yes Yes
Other features Can flex a combination of items through space-occupying features Can push content elements to extreme alignment
Browser Support Firefox, Chrome, Safari, IE10+, IOS Safari Firefox, Chrome, Safari, IE10+, IOS Safari, Opera

(Source: https://www.srijan.net/blog/css-grid-vs-flexbox)

Grid Setup

To set up a div so that its contents are grid elements, we must set the display property to grid, like so:

				                    
				    div {
				        display: grid;
				    }

The display property specifies the display behavior (the type of rendering box) of an element. In HTML, the default display property value is taken from the HTML specifications or from the browser/user default style sheet. The default value in XML is inline, including SVG elements.

(Source: w3schools.com)

Trying to use other display properties on a grid element will have no effect. These properties include:

  • Float
  • Block
  • Inline-block
  • Table-cell

Basic Components

The major elements used to create grids in CSS are grid-gap, grid-template-areas, grid-template-columns, grid-template-rows, and grid-area. Here we will touch on each of these subjects and when they are used.

  • Grid gap - defines the space between grid elements 
    				                        
				grid-gap: 20px;
  • Grid-template-areas - pre-defines the grid within a container by dividing the container into named sections. Grid areas can be repeated if, for instance, you want a row to stretch across multiple columns, as in the bottom row in the example below. 
    				                        
				grid-template-areas:
				    “area1 area2”
				    “area3 area4”
				    “area5 area5”;
  • Grid-area - This property is defined as a style property within the child element. 
    				                            
				child-div {
				    grid-area: area-name;
				}
  • Grid-template-columns - This property is given to a parent element to predetermine how a grid will be divided into columns. 
    				                            
				grid-template-columns: 25% 25% 25% 25%;
  • Grid-template-rows - This property is given to a parent element to predetermine how a grid will be divided into rows. 
    				                            
				grid-template-rows: 50% 50%;

CSS Grid Units

Many units can be used to determine column or row size in Grid. Among those are %, where CSS Grid will adapt a grid item to be that percentage of a container; we can also use traditional, absolute units like px and em.

The new unit type introduced to be used with CSS Grid is the fr unit. The fr unit dictates that a specific fraction will be dedicated to the size of a grid item. The fr unit’s actual size is dependent on the sizes of the other grid items in its container.

If grid-template-rows is given a value of just one 1fr. Then that container is divided into only one grid row and that row takes up 1 fraction of 1 total grid items, or 100% of the space. On the other hand, if it were to be set to 1fr 1fr 1fr, then each grid item these units represent will be 1/3 of the total space, for a total of 100%. The fr unit can also be used in combination with grid items with other units.

The auto unit size stretches the size of the grid item based on whatever space is available.

Justify and Align

As discussed, float and block properties will no longer work with grid items. Therefore, positioning items within a grid is handled differently, with the align and justify properties.

Justify-items and justify-self are properties that apply to the row axis. Justify-items determines how columns behave in a grid container. Justify-self also controls this but can only affect the item it is applied to, and will have no effect on the justification of other grid items.

Align-items and align-self are properties that apply to the column axis. Align-items determines how rows behave, and how they are positioned vertically within a container. Align-self has this same effect but is only applied to a child grid element.

Example Layout

The following is an example of the "holy grail" website layout, a common website design, put together in CSS Grid.

<html>
					<style>
					div { border: 1px solid; }
					body {
					    display: grid;
					    grid-template-columns: 10em auto 10em;
					    grid-template-areas: 
					        "header header header"
					        "left   middle right"
					        "footer footer footer";
					}
					</style>
					<body>
					    <div style="grid-area: header">The header</div>
					    <div style="grid-area: footer">The footer</div>
					    <div style="grid-area: left">The left panel</div>
					    <div style="grid-area: middle; height: 200px">The main content area</div>
					    <div style="grid-area: right">The right panel</div>
					</body>
					</html>
